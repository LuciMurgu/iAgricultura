# Demo Walkthrough

This document provides a click-by-click guide for demonstrating the AgroUnu pilot to a stakeholder.

## 1. Dashboard (`/dashboard`)
- **What to show:** The "Deciziile săptămânii" (Weekly Decisions) list and the "Context fermă" health indicator. Point out the "Configurare fermă" setup card if data is missing.
- **What to say:** "The system is proactive. It doesn't wait for the farmer to run a report; it pushes critical signals to the top. Notice how it explicitly flags missing data instead of hiding it."
- **Risk boundary:** Clarify that these are *signals for human review*, not automated decisions.
- **Client notice:** The lack of horizontal scrolling on mobile and the clean, outcome-focused cards.

## 2. Setup Wizard (`/setup`)
- **What to show:** The 6 outcome paths ("Vreau finanțare", "Vreau să vând mai bine", etc.). Expand one of the forms and show the demo state submission.
- **What to say:** "Onboarding isn't a massive, boring form. It's tied to outcomes. We only ask for the specific context needed to unlock the feature the farmer cares about today."
- **Risk boundary:** Point out the clear "Demo" badges and the "Not an official declaration" disclaimers.
- **Client notice:** The deterministic progress bar that updates instantly without a backend call, proving a solid UI foundation.

## 3. Guided Copilot (`/ask`)
- **What to show:** The guided question templates. Click one to show the preview.
- **What to say:** "We killed the free-text chatbot. Farmers don't want to type paragraphs. They want to click structured questions that guarantee a safe, evidence-based answer."
- **Risk boundary:** Emphasize the blocked actions listed at the bottom (No diagnoses, no eligibility guarantees).
- **Client notice:** The explicit "Necesită verificare umană" (Requires human review) tags on the questions.

## 4. Invoices / Buy Better (`/invoices`)
- **What to show:** The procurement analysis tables and price alerts.
- **What to say:** "This is the most painful area for farmers—input costs. The system compares their invoices against regional signals to flag anomalies."
- **Risk boundary:** State that we do not guarantee savings or endorse specific suppliers.

## 5. Parcels / Fields (`/parcels`)
- **What to show:** The cadastral map representation and crop structure.
- **What to say:** "Agronomy and finance intersect here. We track what is planted where, which is critical for both compliance and yield projections."

## 6. Cooperative (`/cooperative`)
- **What to show:** The network signals and anonymous pooling opportunities.
- **What to say:** "Farmers are hesitant to share data. We show them the value of anonymous regional intelligence first, building trust before asking them to pool resources."

## 7. More (`/more`)
- **What to show:** The secondary navigation menu with all technical ledgers and the disabled "Viitor" (Future) modules.
- **What to say:** "The technical ledgers are still there if needed, but they are tucked away so they don't overwhelm the user's primary workflow."
